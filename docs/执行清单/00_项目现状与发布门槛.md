# 项目现状与发布门槛（你现在离“正式上市”差什么）

## 当前能力（已具备）

- **核心体验**：主页预览、桌面悬浮桌宠、互动动作、商城基础展示、设置与登录页
- **技术底座**：3D 模型加载、动作索引/别名、模型展示参数 profile 化、Release 构建/真机安装
- **账号链路**：微信登录 SDK 接入 + 回调 Activity + 后端换取用户信息（架构已搭好）

## 当前主要短板（按影响优先级）

### P0 阻塞（不解决就无法上架/无法稳定使用）

- **微信开放平台审核未通过**：缺官网、流程图信息不足（当前最大阻塞）
- **登录链路稳定性**：未回调/超时体验差，需要更强的可诊断提示与回归记录
- **合规入口不足**：App 内必须可直达隐私政策/用户协议/联系信息（强烈建议本周补齐）

### P1 必备（能上架但留存/口碑会很差）

- **商城仍是展示态**：缺交易闭环、权益发放、购买恢复
- **缺监控与数据**：崩溃监控、关键埋点、漏斗/留存数据几乎没有
- **缺“新手上手”**：首次进入缺引导与目标，用户容易退出

### P2 增长（决定“像不像流行游戏”）

- 任务/签到/成长线（亲密度/等级）
- 活动与内容更新机制（可运营）
- 账号体系扩展（QQ、游客、绑定/迁移）

---

## 发布门槛（按“流行轻游戏/陪伴类”标准）

### 上架最低门槛（M0 + M1）

- 官网与审核材料一次过
- App 内隐私/协议/联系入口可用
- 核心路径稳定：打开→权限→桌宠→互动→设置→登录→回到首页

### 可商业化门槛（M2）

- 订单/支付/回调验签/幂等/发货/恢复购买闭环
- 客服与退款/争议处理入口

### 可运营门槛（M3 + M4）

- 崩溃监控 + 基础埋点上线
- 每周能发一次版本并可回滚
- 有日常系统驱动留存（任务/签到/成长）
